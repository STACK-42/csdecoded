# The Language of Languages

Before any GPU or CPU was built, before any automaton, before any algorithm, mathematicians needed to agree on what they were even talking about. they kept running into the same problem: how do you precisely describe an infinite collection of strings using a finite set of rules?

if we used natural language here, it fails. Saying "all strings that start with *a* and end with *b*" which is a description of a language, can work for a human, but a machine needs something it can mechanically verify. The notation you are about to learn is important, Every symbol exists because vagueness was the enemy. all is there
is that we want language to describe the language.

## The Three-Level Hierarchy

Everything in [formal language theory](https://en.wikipedia.org/wiki/Formal_language) is built on three concepts, and they are linked! understanding these will lay the foundation for the rest of the course.

### Alphabet ‚Äî $\Sigma$

An alphabet is a **finite, non-empty set of symbols**. 

$$
\Sigma = \{a, b\}
$$

$$
\Sigma = \{0, 1\}
$$

$$
\Sigma = \{x\}
$$

The symbol $\Sigma$ (sigma) is just the conventional name mathematicians chose. The alphabet is the raw material, the only symbols your machine is allowed to see. 
Think of an alphabet as the keys on a keyboard.If your keyboard only has two keys, those are the only characters you can type.

### String (Word)

A string is a **finite sequence of symbols drawn from the alphabet**. You concatenate letters, left to right, and you get a string.

$$
\Sigma = \{a, b\} \implies ab,\ bba,\ aaab\ \text{ are all valid strings}
$$

Two things worth knowing in your mental model right now:

**First** ‚Äî the length of a string $w$ is written $|w|$. So $|abb| = 3$ and $|a| = 1$.

**Second** ‚Äî there is one string that connfuses everyone, and it is the **empty string** $\varepsilon$ (epsilon). It is a string of zero symbols. but be careful, It is not the same as nothing, it exists, it has length zero, and it will appear constantly.

$$
|\varepsilon| = 0
$$

<Callout tone="orange" label="The Œµ mistake">
  The empty string $\varepsilon$ is not the empty set $\emptyset$. The empty set contains nothing. The empty string is a real string ‚Äî it just contains no symbols. A language that contains $\varepsilon$ is different from a language that does not. This distinction will matter in every module that follows.
</Callout>

### Language ‚Äî $L$

A language is simply a **set of strings** over some alphabet. That's it.

$$
L = \{ab,\ ba,\ aba\}
$$

$$
L = \{w \in \Sigma^* \mid w \text{ starts with } a\}
$$

Both of those are valid languages. One is finite and listed explicitly. The other is infinite and described by a rule. Formal language theory is almost entirely about finding compact, mechanical rules to describe infinite languages ‚Äî because you cannot list them explicitly.

## Kleene Star ‚Äî $\Sigma^*$

This is the most important operator in the course. You will see it everywhere.

$\Sigma^*$ means **all possible strings over $\Sigma$, including the empty string**. Every string of every length, formed from any combination of symbols in the alphabet.

$$
\Sigma = \{a, b\} \implies \Sigma^* = \{\varepsilon,\ a,\ b,\ aa,\ ab,\ ba,\ bb,\ aaa,\ aab,\ \ldots\}
$$

$\Sigma^*$ is always infinite (unless $\Sigma$ itself is empty). It is the universe of all strings your alphabet can produce.

The $+$ variant ‚Äî $\Sigma^+$ ‚Äî is the same thing but **excluding** $\varepsilon$:

$$
\Sigma^+ = \Sigma^* - \{\varepsilon\}
$$

So every language $L$ you will ever study is a subset of $\Sigma^*$:

$$
L \subseteq \Sigma^*
$$

And that sums the essence of the whole hierarchy. A language is just a subset of the universe of possible strings.

<Callout tone="violet" label="Why Œ£* exists">
  Think of $\Sigma^*$ as your entire string universe. Without it, you would have to define what a valid string is every single time you create a language. With it, you just write $L \subseteq \Sigma^*$, and everyone instantly knows your language is a piece of that universe.
</Callout>


## Concatenation

Concatenation is the operation of joining two strings end-to-end. If $u = ab$ and $v = ba$, then:

$$
uv = abba
$$

This extends to languages. The **concatenation of two languages** $X$ and $Y$ is:

$$
XY = \{uv \mid u \in X \text{ and } v \in Y\}
$$

Every string from $X$ paired with every string from $Y$, joined together. If $X = \{a,\ bb\}$ and $Y = \{c,\ dd\}$:

$$
XY = \{ac,\ add,\ bbc,\ bbdd\}
$$

Exponents abbreviate repeated concatenation:

$$
a^3 = aaa \qquad \Sigma^2 = \{aa,\ ab,\ ba,\ bb\}
$$

And $w^0 = \varepsilon$ for any string $w$ ‚Äî concatenating zero times gives you the empty string.

## Regular Expressions ‚Äî The First Compact Notation

You may have heared Regular expressions, probably from `grep`. but they are not just that, They are a formal mathematical notation for describing [regular languages](https://en.wikipedia.org/wiki/Regular_language), invented by [Stephen Kleene](https://en.wikipedia.org/wiki/Stephen_Cole_Kleene) in 1951, and based on exactly the three operations you just learned.

A regular expression over $\Sigma$ is defined by three rules:

**Basis** ‚Äî these are the atoms:
- $\varepsilon$ is a regular expression, describing the language $\{\varepsilon\}$
- $\emptyset$ is a regular expression, describing the empty language $\{\}$
- Any single symbol $a \in \Sigma$ is a regular expression, describing $\{a\}$

**Recursion** ‚Äî if $r_1$ and $r_2$ are regular expressions, so are:
- $r_1 r_2$ ‚Äî concatenation
- $r_1 + r_2$ ‚Äî union (either $r_1$ or $r_2$)
- $r_1^*$ ‚Äî Kleene star (zero or more repetitions of $r_1$)

**Closure** ‚Äî nothing else is a regular expression.

Those three rules generate every regular expression that has ever been written. The regex in your editor, the pattern in your validation code, the search in your terminal ‚Äî all of it is an instance of this recursive definition.

A worked example to make this concrete. Given $\Sigma = \{a, b\}$:

$$
L = \text{language}(a^*b^*) = \{\varepsilon,\ a,\ b,\ aa,\ ab,\ bb,\ aaa,\ aab,\ abb,\ bbb,\ \ldots\}
$$

All strings of zero or more $a$'s followed by zero or more $b$'s. The regular expression $a^*b^*$ describes this infinite language in five characters.

$$
L = \text{language}((a+b)^*) = \Sigma^*
$$

Every possible string over $\{a, b\}$. The $(a+b)^*$ pattern ‚Äî which you might have seen in real regex ‚Äî is just the formal way of writing the entire universe of strings.

> **Note:** Throughout this course, $\Sigma = \{a, b\}$ and $\Sigma = \{0, 1\}$ are interchangeable examples ‚Äî both are just two-symbol alphabets. Do not read meaning into the symbols themselves. $a$ and $b$ are placeholders, exactly like $0$ and $1$.

## Quick Reference

| Symbol         | Name              | Means                                                    |
| -------------- | ----------------- | -------------------------------------------------------- |
| $\Sigma$       | Alphabet          | Finite, non-empty set of symbols                         |
| $\varepsilon$  | Empty string      | A string of zero symbols ($\lvert\varepsilon\rvert = 0$) |
| $\emptyset$    | Empty set         | The set containing no strings                            |
| $\Sigma^*$     | Kleene star       | All strings over $\Sigma$, including $\varepsilon$       |
| $\Sigma^+$     | Positive closure  | All strings over $\Sigma$, excluding $\varepsilon$       |
| $L$            | Language          | A subset of $\Sigma^*$                                   |
| $XY$           | Concatenation     | $\{uv \mid u \in X,\ v \in Y\}$                          |

<Callout tone="mint" label="What you just built">
  You now have the three primitives that everything else in this course rests on: an alphabet defines the symbols, strings are finite sequences of those symbols, and languages are sets of strings. Regular expressions are the first tool for describing those sets compactly. Every machine you study from here ‚Äî finite automata, pushdown automata, Turing machines ‚Äî is just a different mechanism for recognizing which strings belong to a language and which do not.
</Callout>
